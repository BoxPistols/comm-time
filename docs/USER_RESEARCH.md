| **勤務形態** | 週5日在宅勤務 |
| **家族構成** | 妻と2歳の子供 |
| **デバイス** | MacBook Pro（業務用）、iPhone 14、iPad |
| **技術リテラシー** | 高い |

#### 背景・状況
- スタートアップ企業でフルリモート勤務
- 1日平均3〜5回のオンラインミーティング
- Slack、Zoom、Google Meetを頻繁に使用
- 自宅で仕事と育児を両立している

#### 課題・ペインポイント
- ミーティングが予定時間を超過しがち

- ミーティングを時間通りに終わらせたい
- 効率的に集中時間を確保したい
- 会議での決定事項を確実に記録したい
- シンプルで使いやすいツールを求めている

#### 利用シーン
- 日中のスタンドアップミーティング（15分）
- 週次の1on1ミーティング（30分）
- プロジェクトの進捗会議（60分）
- コーディング作業中のポモドーロ

#### 期待する価値
> 「会議を時間内に終わらせて、その分コーディングに集中したい。シンプルなツールで全部管理できたら最高」

## 2. カスタマージャーニーマップ

### 2.1 佐藤健太のジャーニー（リモートワーカー）

```
フェーズ:   認知 → 検討 → 導入 → 習慣化 → 推奨
```

| フェーズ | ユーザーの行動 | 思考・感情 | タッチポイント | 課題・機会 |
|---------|---------------|-----------|---------------|-----------|
| **認知** | 「会議 タイマー アプリ」で検索 | 「会議が長引いてイライラ...」😤 | Google検索、Qiita記事 | SEO対策、技術記事での紹介 |
| **興味** | Comm Timeのサイトを訪問 | 「シンプルでいい感じ」🤔 | ランディングページ | 機能の魅力を伝えるコピー |
| **検討** | 無料で試せることを確認 | 「登録不要で使えるのがいい」😊 | デモサイト | 即座に試せる体験 |
| **導入** | 初回利用、タイマー設定 | 「直感的で使いやすい！」😃 | アプリUI | オンボーディング体験 |
| **活用** | 毎日のスタンドアップで使用 | 「会議が時間通りに終わる」🎉 | デイリー使用 | 継続利用のモチベーション |
| **習慣化** | ポモドーロも活用開始 | 「集中力が上がった」💪 | 複数機能の利用 | 機能の発見・活用促進 |
| **推奨** | チームメンバーに紹介 | 「みんなにも使ってほしい」🙌 | Slack、口コミ | 共有機能、招待機能 |

#### タッチポイント詳細

```
[認知段階]
    │
    ├─ Google検索「会議 タイマー オンライン」
    ├─ Qiita/Zenn技術記事
    ├─ Twitter/X での推薦ポスト
    └─ 同僚からの口コミ
    │
    ▼
[興味段階]
    │
    ├─ ランディングページ訪問
    ├─ 機能一覧の確認
    ├─ スクリーンショット閲覧
    └─ 「登録不要」の発見
    │
    ▼
[導入段階]
    │
    ├─ 即座にタイマー開始
    ├─ アラームポイント設定
    ├─ 音量・通知のカスタマイズ
    └─ 初回ミーティングで使用
    │
    ▼
[活用段階]
    │
    ├─ 毎日のスタンドアップ
    ├─ TODO機能の発見
    ├─ メモ機能の活用
    └─ ポモドーロモードの発見
    │
    ▼
[習慣化・推奨段階]
    │
    ├─ クラウド同期でアカウント登録
    ├─ 複数デバイスで利用
    ├─ チームへの推奨
    └─ フィードバック送信
```

### 4.3 主要な発見事項

#### 🔍 発見1: 会議の時間超過は普遍的な課題

**インタビュー結果:**

> 「1日5回以上のミーティングがあって、1つ延びると全部ずれる。本当にストレス」
> — U3（42歳、シニアPM）

> 「スタンドアップは15分と決めてるのに、気づいたら30分経ってることがよくある」
> — U1（29歳、バックエンドエンジニア）

**定量データ:**
- 8名中7名が「週に3回以上」会議が予定時間を超過すると回答
- 平均超過時間: 15〜20分
- 超過の主な原因: 「時間を意識していない」「話が盛り上がる」

**示唆:**
- 視覚的な時間表示のニーズが高い
- 残り時間のアラート機能が有効

---

#### 🔍 発見2: シンプルさが最重要

**インタビュー結果:**

> 「既存のツールは機能が多すぎて、結局使わなくなった」
> — U7（48歳、イラストレーター）

> 「すぐ使い始められるのが一番大事。登録とか設定とか面倒」
> — U5（20歳、大学2年生）

**定量データ:**
- 8名中6名が「シンプルさ」を最優先と回答
- 過去に使用したツールを辞めた理由の1位: 「複雑すぎる」

**示唆:**
- 登録不要で即座に使える体験が重要
- 機能追加は慎重に、シンプルさを維持

---

#### 🔍 発見3: モバイル対応の必要性

**インタビュー結果:**

> 「外出先でスマホからも同じ設定で使いたい」
> — U4（36歳、スクラムマスター）

> 「カフェで勉強するときはスマホがメイン」
> — U6（23歳、大学院生）

**定量データ:**
- 8名中5名が「スマートフォンでも使いたい」と回答
- 複数デバイスで利用したい: 75%

**示唆:**
- レスポンシブデザインは必須
- クラウド同期機能の価値が高い

---

#### 🔍 発見4: 視覚的アラートへの期待

**インタビュー結果:**

> 「音だけだと会議中は気づかない。画面が光るとか視覚的なのがいい」
> — U3（42歳、シニアPM）

> 「ヘッドフォンしてると音が聞こえないことがある」
> — U2（35歳、デザインエンジニア）

**定量データ:**
- PMの4名中4名が視覚的アラートを希望
- 希望するアラート方法: 画面フラッシュ(75%)、色の変化(50%)、バイブレーション(37.5%)

**示唆:**
- フラッシュアラート機能は差別化ポイント
- 複数のアラート方法を組み合わせる

---

#### 🔍 発見5: TODOとメモの一元管理

**インタビュー結果:**

> 「会議のメモとTODOが別々のツールだと、後で紐付けが面倒」
> — U4（36歳、スクラムマスター）

> 「その場でアクションアイテムを書いて、すぐにタスク化したい」
> — U1（29歳、バックエンドエンジニア）

**定量データ:**
- 8名中6名が「時間管理とタスク管理を一緒にしたい」と回答

**示唆:**
- タイマー + TODO + メモの統合に価値あり
- ツール乗り換えコストを下げる
